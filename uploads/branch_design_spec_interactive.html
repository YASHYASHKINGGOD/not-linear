Interactive design spec for Branch — a non-linear AI note-taking canvas

Design spec — v0.1

Branch

A canvas for non-linear thinking. Notion's quiet surface, Claude's depth, Whimsical's spatial logic — collapsed into a single tool. Below is the spec, with the core interactions live so you can feel them.

01
The branch gesture

Highlight any text. The popover appears. "Branch" is the primary action — same muscle memory as Notion's comment, just pointed at a different verb.

Try it  Select text in the doc below.

Live demo — text selectionSelection → popover → primary action: Branch

Focused Chat View

  • Single-column layout when reading or typing inside a node — same rhythm as a normal chat.
  • User bubbles right-aligned with a warm tint; assistant prose runs full-width and renders markdown.
  • Source quote breadcrumb at the top shows which selection triggered this branch, with a back button to the canvas.
  • Per-node model selection sits in the composer: Sonnet for the heavy thinking, a cheaper model for follow-ups.

Why this works

01

Borrowed muscle memory

Notion users already select text and reach for a popover. We don't teach a new gesture — we re-aim an existing one.

02

Source becomes context

The selected phrase is pinned at the top of the new node. The branch always knows what it came from.

03

One primary verb

Branch leads. Comment, format, and AI edits live in the same bar but visually recede so the canvas action stays obvious.

04

No menu diving

Two clicks from any sentence to a new line of thinking. If it takes more, the gesture has failed.

02
The canvas

Every node is a chat. Edges show ancestry. The dot grid is your only background — no gradients, no decoration. Click a node to enter focused view; click the canvas to back out.

03
Focused chat

Inside a node, the canvas disappears. You're in a familiar chat — just with a breadcrumb trail at the top showing where you came from, and the same selection-to-branch gesture available on every assistant reply.

What's the impact of single-column layout in a node where users might want to compare two responses side by side?
Single-column is a deliberate trade. Inside a node you're deep in one line of thinking — comparison is the canvas's job, not the chat's. If you want two responses next to each other, branch them as siblings and let the canvas show them at the same zoom level. Forcing split panes inside a node would recreate the cognitive clutter the tool exists to fix.
So branching is the primary muscle, not pane-splitting. Got it.
Right. Pane-splitting is a workaround people invented because their tool wouldn't let them hold multiple threads as first-class objects. Once threads are first-class — nodes on a canvas — the need for split panes evaporates.

05
Anti-patterns
Don'tWhy
Multi-pane chat inside a nodeIf you want comparison, that's what the canvas is for. Branch as siblings.
Tree icons or folder metaphorsThis isn't a filesystem. The graph is the metaphor; don't dilute it with a sidebar tree.
Purple, gradients, glass blurGeneric AI aesthetic. The tool earns trust by looking like a tool, not a product launch.
Right-rail sidebars on canvasThe canvas needs all the room. Properties go in a bottom-sheet on node selection.
Auto-layout that re-arranges nodesSpatial memory is half the value. Never move a node the user didn't move.
Notifications, badges, streaksThis is a thinking tool. It should never demand attention.
